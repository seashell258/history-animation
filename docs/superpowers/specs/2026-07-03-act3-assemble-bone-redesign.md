# 第三幕主互動重設計：拼碎骨讀裂紋

**Date:** 2026-07-03
**Supersedes:** 原 spec §4「第三幕：三選一派將」（2026-07-02-shang-dynasty-interactive-design.md）
**Scope:** 只改第三幕主互動場景（`act3-choose` → `act3-assemble`）。第一/二/四幕不動。

## 1. 為什麼要改

現行 `act3-choose` 讓學生從三個將領剪影任選其一，無論選誰都揭曉「其實是婦好」。實作上滿足了原 spec §3「選擇僅製造參與感，不設懲罰」的原則，但實際玩起來：

- 選擇不影響任何後果，學生會發現「選了等於沒選」
- 中間位置強制是答案，感覺劇本綁架體驗
- 「三選一」被降格為過場點擊，沒創造記憶點

歷史事實不能動（婦好帶兵是甲骨鐵證）。要做的是把**互動的意義**從「你決定派誰」平移到「你親手讀出甲骨的答案」——由學生把燒炸開的甲骨拼回去，讓裂紋自己指出人選。

## 2. 設計目標

- **保留學生參與感**，但移除「選擇即結果」的假選項
- **強化占卜文化的直觀感**：甲骨會說話、王在讀裂紋，而不是在挑人
- **提供一次「拼完 → 顯形」的雙重揭曉**（方向 + 身分）作為記憶點
- **不重複其他幕的機制**：不是按住燒（act1）、不是拖字塊配對（act2）、不是滑動磨擦（act4）；是**幾何拼合＋雜訊釋讀**

## 3. 新機制：拼碎骨讀裂紋

### 3.1 幕內流程

```
[act3-intro]  狼煙 + 警報閃 + 鼓聲                （不變）
   ↓ 「立刻問天 →」
[burn]        按住龜甲 1.6 秒，提示「按住龜甲——問這一仗」    （不變）
   ↓ 燒完 → 裂
[act3-assemble]  4 片碎骨飄開，隨機分布            （新，取代 act3-choose）
   學生拖 4 片回中央虛框 → 吸附咔                （新）
   全部到位 → 雜訊裂紋歸攏成向左偏斜的主脈       （新）
   → 左邊剪影填色亮起：婦好顯形 + 名字白卡        （新）
   ↓ 「出征！ →」
[act3-verify]  驗辭「後來應驗了嗎——勝」          （不變，Task 7 才做）
```

現行 `#scene-act3-choose` section 的 HTML/CSS/JS **整段汰除**，場景 id 直接改名為 `act3-assemble`（`section[data-scene]`、`#scene-*` id、`SCENE_ORDER` 陣列、`burn` 的 setBurn target 全部一起改）。章節圓點導覽仍指向 `act3-intro`，不受影響。

### 3.2 場景組成

**背景**：深墨底（同現行 act3-choose 色調），無城牆狼煙（那是 intro 專屬）。

**三個剪影**：位置固定
- 左：婦好——**進場時是無臉黑影**（跟另兩位視覺完全一致）
- 中：無臉黑影（男將軍 A）
- 右：無臉黑影（男將軍 B）

三個剪影一開始看不出任何差異，全都是同尺寸同姿態的黑色人形（略帶肩甲輪廓即可）。**婦好的識別特徵（髮髻、硃砂紅戈、金光）在被裂紋指中的那一瞬才顯形**。

**中央虛框**：甲骨原本的位置，虛線輪廓（淺色描邊，`stroke-dasharray`），提示「拼回這裡」。

**4 片碎骨**：進場時從中央「炸」開，飄浮在畫面環狀範圍內。每片：
- 有各自預定的槽位（拼合座標／旋轉）
- 表面帶多分岔的細裂痕紋理（**看起來像隨機噪音**，實際上每片藏著主脈的一小段）

### 3.3 互動細節

**碎片起始位置**：每次進場**隨機**（限制在一個以中央為圓心、r ≈ 250–320 的環狀範圍內，且不與任一剪影重疊）。旋轉角度也可隨機。

**抓取**：滑鼠按下（或 touchstart）在碎片上開始拖動。碎片提到最上層，指標跟手。

**吸附閾值**：當碎片中心與其目標槽位距離 < ~60px 時，放手觸發吸附——
- 播 `pop` 音
- 位置與旋轉插值到正確值（~200ms 動畫）
- 鎖定，之後不可再拖

**拖錯**（放手時不在閾值內）：
- 碎片彈回原本飄浮位置（~300ms 緩衝動畫）
- 不扣分、不提示錯誤，繼續飄著等再試

**指標離開畫面／pointerup 未捕捉**：視同「拖錯」——碎片彈回原飄浮位置。用 `pointercancel` 與畫面外 `pointerup` 兜底，避免碎片黏在指標上。

**順序**：任意。4 片各有各的槽位，互相獨立。

**裂紋顯現**：
- 碎片吸附後，其表面的裂紋紋理原地保留（就跟碎片本身一起顯示）
- 前 2 片：裂痕看起來雜亂、無明確方向
- 第 3 片：主脈開始隱約可見（但仍被許多支脈干擾）
- **第 4 片吸附完成的瞬間**（動畫收尾）：
  - 所有支脈／雜訊線條淡出（~400ms）
  - 主脈——一條向左偏斜的鋸齒狀裂紋——**亮起金色**（stroke 顏色從 crack 深色補間到 gold）
  - 播 `ding`

### 3.4 揭曉腳本

主脈亮起約 300ms 後：
1. 左邊剪影開始「填色動畫」：黑影 → 逐步浮出髮髻（頭頂彎鉤形狀）、硃砂紅戈（斜挎腰間）、身形描邊金光
2. 剪影上方或畫面正中偏上浮出白卡（複用 `#fuhao-reveal` 的視覺樣式）：
   - 上排小字：「這位——」（或空）
   - 大字：**婦好**（88px 硃砂色）
   - 下排：「商王的王后——也是帶兵打仗的**女將軍**。<br>這不是編的：甲骨上真的刻著，她帶了**一萬三千人**出征。」
   - 下方：`出征！ →` 按鈕，`game.goto('act3-verify')`

**中／右剪影維持黑影**（強化對比：這麼多將軍裡，答案是最左邊那位——王后）。

### 3.5 邊角情況

**重入本場景**（章節圓點跳回、或劇情倒退）：
- 4 片碎骨重新飄回散開位置（重新隨機）
- 剪影還原黑影
- 主脈裂紋、白卡淡出
- 邏輯與現行 act3-choose reset 一致：監聽 `scene:enter` 事件，`e.detail.scene === 'act3-assemble'` 時執行 reset

**音效**：
- 吸附成功：`audio.pop()`（沿用）
- 主脈亮起 + 揭曉：`audio.ding()`（沿用）
- 進場鼓聲不變（`drumRoll` 由 intro 觸發，本場景不再觸發）

**無 idle hint**：4 片飄在畫面上、中央虛框明示目的地，資訊已足夠；不加超時提示避免囉唆。

**Accessibility**：碎片支援鍵盤操作暫不做（此互動屬 pointer/touch first；整個作品已假設有滑鼠或觸控）。

## 4. 不變的東西（避免誤動）

- `act3-intro` 場景（狼煙、警報、鼓聲、`立刻問天` 按鈕）——**維持現行實作**
- `audio.drumRoll` 方法——**維持**（intro 觸發）
- `burn` 場景與 `setBurn('act3-choose', 1.6, ...)` 呼叫——**目標場景 id 需改成 `act3-assemble`**
- `act3-verify` 場景（Task 7 才做，本次不動）
- 章節圓點導覽（既有的 `act3-intro` target 就是入口）
- selftest 17/17 全數維持通過

## 5. 待廢除

- `#scene-act3-choose` section 整段 HTML
- `.choose-title / #generals / .general / #fuhao-reveal / #fuhao-line1 / #fuhao-name / #fuhao-sub` 相關 CSS
- 「三選一」JS：`FU_HAO`、`.general` click handler、`act3-choose` 的 `scene:enter` reset、`act3-verify-btn` click handler（後兩者的**行為**保留、但綁到新的元素 id 上）
- 場景 id 從 `act3-choose` 改為 `act3-assemble`；`SCENE_ORDER`、章節圓點、`burn` target 相應更新
